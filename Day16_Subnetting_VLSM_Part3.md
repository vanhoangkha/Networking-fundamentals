# NGÀY 16: SUBNETTING VLSM PART3 ## 16.1 QUÁ TRÌNH CHIA MẠNG CON Quá trình chia mạng con Class A, Class B, và Class C là giống hệt nhau. ## 16.2 CHIA MẠNG CON CLASS A **Cho mạng 10.0.0.0/8**, bạn phải tạo 2000 mạng con để phân phối cho các doanh nghiệp khác nhau. Bạn phải sử dụng prefix length nào? **Tính toán:** - 2^10 = 1024 (không đủ) - 2^11 = 2048 (đủ) Chúng ta phải "mượn" 11 bit (từ trái sang phải) để có đủ mạng con: ``` 0000 1010. 0000 0000. 000|00000. 0000 0000 8 bit + 8 bit + 3 = 19 bit ``` **mạng con mask:** ``` 1111 1111. 1111 1111. 111|00000. 0000 0000 255.255.224.0 ``` **Đáp án là /19** (/8 + /11 = /19) **Số host trên mỗi mạng con:** Còn lại 13 bit host: 2^13 - 2 = **8190 host trên mỗi mạng con** **Đặc điểm:** ## 16.3 VARIABLE - LENGTH mạng con MASKS (VLSM) ## 16.4 # So sánh FLSM vs VLSM: **FLSM (Fixed - Length mạng con Masks):** - Tất cả mạng con sử dụng cùng prefix length - Ví dụ: Chia mạng Class C thành 4 mạng con sử dụng /26 **VLSM (Variable - Length mạng con Masks):** - Quá trình tạo các mạng con có kích thước khác nhau - Sử dụng địa chỉ mạng hiệu quả hơn - Phức tạp hơn FLSM nhưng dễ nếu làm theo đúng các bước **Đặc điểm:** ## 16.5 VÍ DỤ THỰC HÀNH VLSM ![image](HTTPS: //github.com/psaumur/CCNA/assets/106411237/30a08f93 - TO - POINT CONNECTION** (giữa hai router) ## 16.7 # Mạng gốc: 192.168.1.0/24 ``` 192.168.1.0/24 11000000.10101000.00000001|00000000 (octet cuối là host = 254 host sử dụng được) ``` **Nguyên tắc:** - Dịch TRÁI → TĂNG ĐÔI số host - Dịch PHẢI → GIẢM NỬA số host **Đặc điểm:** ## 16.8 BƯỚC 1: TOKYO LAN (Local Area Network) A (110 HOST) Cần mượn 1 bit host sang phải để có 2^7 = 128 host (đủ cho 110 host): ``` 192.168.1.0/25 11000000.10101000.00000001.0|0000000 Chuyển bit host còn lại thành 1: 01111111 = 127 Broadcast: 192.168.1.127/25 ``` **Kết quả TOKYO LAN (Local Area Network) A:** - **Network:** 192.168.1.0/25 - **Broadcast:** 192.168.1.127/25 - **First usable:** 192.168.1.1/25 - **Last usable:** 192.168.1.126/25 - **Tổng host sử dụng được:** 126 (2^7 - 2) **mạng con tiếp theo bắt đầu từ:** 192.168.1.128 **Đặc điểm:** ## 16.9 BƯỚC 2: TORONTO LAN (Local Area Network) B (45 HOST) Cần 2^6 = 64 host (đủ cho 45 host): ``` 192.168.1.128/26 11000000.10101000.00000001.10|000000 Chuyển bit host thành 1: 10111111 = 191 Broadcast: 192.168.1.191/26 ``` **Kết quả TORONTO LAN (Local Area Network) B:** - **Network:** 192.168.1.128/26 - **Broadcast:** 192.168.1.191/26 - **First usable:** 192.168.1.129/26 - **Last usable:** 192.168.1.190/26 - **Tổng host sử dụng được:** 62 (2^6 - 2) **mạng con tiếp theo bắt đầu từ:** 192.168.1.192 **Đặc điểm:** ## 16.10 BƯỚC 3: TORONTO LAN (Local Area Network) A (29 HOST) Cần 2^5 = 32 host (đủ cho 29 host): ``` 192.168.1.192/27 11000000.10101000.00000001.110|00000 Chuyển bit host thành 1: 11011111 = 223 Broadcast: 192.168.1.223/27 ``` **Kết quả TORONTO LAN (Local Area Network) A:** - **Network:** 192.168.1.192/27 - **Broadcast:** 192.168.1.223/27 - **First usable:** 192.168.1.193/27 - **Last usable:** 192.168.1.222/27 - **Tổng host sử dụng được:** 30 (2^5 - 2) **mạng con tiếp theo bắt đầu từ:** 192.168.1.224 **Đặc điểm:** ## 16.11 BƯỚC 4: TOKYO LAN (Local Area Network) B (8 HOST) Cần 2^4 = 16 host (đủ cho 8 host): ``` 192.168.1.224/28 11000000.10101000.00000001.1110|0000 Chuyển bit host thành 1: 11101111 = 239 Broadcast: 192.168.1.239/28 ``` **Kết quả TOKYO LAN (Local Area Network) B:** - **Network:** 192.168.1.224/28 - **Broadcast:** 192.168.1.239/28 - **First usable:** 192.168.1.225/28 - **Last usable:** 192.168.1.238/28 - **Tổng host sử dụng được:** 14 (2^4 - 2) **mạng con tiếp theo bắt đầu từ:** 192.168.1.240 **Đặc điểm:** ## 16.12 BƯỚC 5: POINT -
