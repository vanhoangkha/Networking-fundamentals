# 7. ĐỊA CHỈ IPV4 : PART 1 OSI MODEL - Mạng LAYER (Layer 3) - Provides connectivity between end hosts trên DIFFERENT networks (ie: outside của the LAN) - Provides logical addressing (IP addresses) - Provides path selection between SOURCE và DESTINATION - ROUTERS operate tại LAYER 3 Định tuyến SWITCHES (Layer 2 Devices) do no separate different networks. They connect và EXPAND networks within the same LAN. By adding một Router, however, between two SWITCHES, you create một SPLIT trong the Mạng; each với it's own Mạng Địa chỉ IP. Example: 192.168.1.0/24 (255.255.255.0) 192.168.2.0/24 (255.255.255.0) ![image](https://github.com/psaumur/CCNA/assets/106411237/3d414956-cb53-46f6-b386-3fc9bba11802) ROUTERS have unique IP Addresses cho EACH của their Giao diện connections, depending trên their location. The Địa chỉ IP cho the Router's G0/0 Giao diện is: 192.168.1.254/24 The Địa chỉ IP cho the Router's G0/1 Giao diện is: 192.168.2.254/24 ![image](https://github.com/psaumur/CCNA/assets/106411237/6e593774-4113-4493-89bb-4d394cb29e1d) The Địa chỉ IP depends trên Mạng Địa chỉ của the LAN it là connects to. The Mạng portion của given Địa chỉ IP sẽ be the same cho all HOSTS trên một given LAN. Example: 192.168.1.100 192.168.1.105 192.168.1.205 All của these addresses là trên the SAME Mạng because the Mạng PORTION của their Địa chỉ IP là the same (192.168.1) while the HOST part (100,105,205) is UNIQUE! When một Broadcast message hits một Router, it does NOT continue onward. It stays within the LOCAL LAN (Switch/Hosts). --- IPv4 Header ![image](https://github.com/psaumur/CCNA/assets/106411237/4f4bd7da-1876-4000-8229-be4b8792a86d) IP (or Internet Giao thức) is the primary Layer 3 Giao thức trong use today. Version 4 là the version trong use trong most networks. IPv4 Headers contain MORE fields than the Ethernet Header. IPv4 Headers contain một SOURCE Địa chỉ IP và DESTINATION Địa chỉ IP field. This FIELD là 32-bits(4-bytes) in length (0-31) 192.168.1.254 (each decimal number represents 8 bits) Translated đến Binary: 11000000 . 10101000 . 00000001 . 11111110 EACH của these 8 bit groups là referred đến as một OCTET Since Binary là difficult đến read cho people, we use the Dotted Decimal format. --- REVIEW của DECIMAL và HEXADECIMAL ![image](https://github.com/psaumur/CCNA/assets/106411237/e45f0ea9-a705-463b-bb9b-d81034cf130d) Decimal (base 10) Ex: 3294 = (3 * 1000) + (2 * 100) + (9 * 10) + (4 * 1) Hexadecimal (base 16) Ex: 3294, would be CDE ``` C (C * 256 / 12 * 256 = 3072) // 256ths position D (D * 16 / D=13 so 16*13 = 208) // 16ths position E (E * 1 / E = 14) // 1s position ``` Adding these up, we get 3294 --- So, how do we convert một BINARY NUMBER đến một DECIMAL NUMBER? The same way we convert đến Hexadecimal. 10001111 So: ``` 1 * 128 = 128 1 * 8 = 8 1 * 4 = 4 1 * 2 = 2 1 * 1 = 1 ``` Add them all up : 128 + 8 + 4 + 2 + 1 = 143 The answer là 143. --- Another example: 01110110 ``` 1 * 64 = 64 1 * 32 = 32 1 * 16 = 16 1 * 4 = 4 1 * 2 = 2 ``` Add them all up: 64 + 32 + 16 + 4 + 2 = 118 The answer là 118. --- Another example: 11101100 ``` 1 * 128 = 128 1 * 64 = 64 1 * 32 = 32 1 * 8 = 8 1 * 4 = 4 ``` Add them all up: 128 + 64 + 32 + 8 + 4 = 236 The answer là 236. --- So, how do we convert một DECIMAL NUMBER đến một BINARY NUMBER? Take the number 221. We có thể take that number và start subtracting it từ LEFT đến RIGHT của our Binary slots. 221 ``` 221 - 128 = 93 so we place a 1 in the "128" slot ``` 10000000 ``` 93 - 64 = 29 so we place another 1 in the "64" slot 29 - 32 isn't possible so we place a 0 in the "32" slot 29 - 16 = 13 so we place a 1 in the "16" slot 13 - 8 = 5 so we place a 1 in the "8" slot 5 - 4 = 1 so we place a 1 in the "4" slot 1 - 2 isn't possible so we put a 0 in the "2" slot 1 - 1 is possible so we put a 1 in the "1" slot ``` This, then, allows us đến the write out the BINARY number cho 221. It is : 11011101 --- Another example: 127 ``` 127 - 128 is not possible so 0 in "128" 127 - 64 is possible so 1 in "64" 63 - 32 is possible so 1 in "32" 31 - 16 is possible so 1 in "16" 15 - 8 is possible so 1 in "8" 7 - 4 is possible so 1 in "4" 3 - 2 is possible so 1 in "2" 1 is possible so 1 in "1" ``` So 127, in BINARY, is 0111 1111 --- Another example: 207 Alternatively, you có thể subtract the number from '255' (which là 1111111). The remainder, then, can be used to "find" where the 0's là trong the binary number. 255 - 207 = 48 so ... 1 1 0 0 1 1 1 1 (32 + 16 = 48) 11001111 là the correct answer. --- IPv4 ADDRESSES So we now know that IP Addresses là the Dotted Decimal conversion của một series của BINARY NUMBERS (broken up into 4 OCTETS) like so: 192.168.1.254/24 But what does the /24 stand for? ![image](https://github.com/psaumur/CCNA/assets/106411237/808fa7fa-0239-42fa-9706-79db87ea167e) It means the FIRST 24 BITS của this Địa chỉ represent the Mạng portion của the Địa chỉ. 192.168.1 là the Mạng PORTION (the first 3 OCTETS) .254 là the HOST PORTION (the last OCTET) ![image](https://github.com/psaumur/CCNA/assets/106411237/2e7c64e1-5689-486a-bba0-9623f5e8de7d) --- CONVERT this BINARY number into một IPv4 Địa chỉ: 10011010010011100110111100100000 10011010 . 01001110 . 01101111 . 00100000 Octets: 1. 128 + 16 + 8 + 2 = 154 2. 64 + 8 + 4 + 2 = 78 3. 64 + 32 + 8 + 4 + 2 + 1 = 111 4. 32 The IPv4 Địa chỉ is: 154.78.111.32/16 154.78 là the Mạng PORTION 111.32 là the HOST PORTION Another Example: 00001100100000001111101100010111 00001100 . 10000000 . 11111011 . 00010111 Octets: 1. 8 + 4 = 12 2. 128 3. 255 - 4 = 251 4. 16 + 4 + 2 + 1 = 23 The IPv4 Địa chỉ is: 12.128.251.23/8 12 là the Mạng PORTION 128.251.23 là the HOST PORTION --- IPv4 Địa chỉ CLASSES IPv4 ADDRESSES là split up into 5 different 'classes'. The class của một IPv4 là determined bởi the FIRST OCTET của the Địa chỉ. CLASS FIRST OCTET FIRST OCTET NUMBERIC RANGE A 0xxxxxxx 0-126 + 127 'loopback' B 10xxxxxx 128-191 C 110xxxxx 192-223 D 1110xxxx 224-239 E 1111xxxx 240-255 From the above chart, if the FIRST OCTECT STARTS với 0, the numeric RANGE của possible first DOTTED DECIMAL là between 0-127. The CLASSES we sẽ be focusing trên là CLASS một đến CLASS C. ![image](https://github.com/psaumur/CCNA/assets/106411237/7cc286bf-ce76-4eee-af52-062a63dac2b4) D CLASS là reserved for 'Multicast' ADDRESSES E CLASS là reserved for 'EXPERIMENTAL' ADDRESSES --- A CLASS USUALLY have một range của 1-126? WHY? Because 127 là usually reserved for 'loopback addresses' 127.0.0.0 đến 127.255.255.255 là used đến test the Mạng. - Used đến test the 'Mạng stack' (OSI & TCP/IP model) on the local Thiết bị. --- ![image](https://github.com/psaumur/CCNA/assets/106411237/25f7db1a-f934-4c73-9926-66bb207fd292) The PREFIX LENGTH là the LENGTH của the Mạng PORTION của the Địa chỉ. From the examples above: 12.128.251.23/8 là một CLASS một Địa chỉ 154.78.111.32/16 là một CLASS B Địa chỉ 192.168.1.254/24 là một CLASS C Địa chỉ Because the Mạng portion của CLASS một là so short, it means there là một LOT more potential Hosts. Because the Mạng portion của CLASS C là so long, it means fewer potential Hosts. --- NETMASK ![image](https://github.com/psaumur/CCNA/assets/106411237/874c022f-9b8c-4862-a495-597682b014a4) A NETMASK là written like một Dotted Decimal Địa chỉ IP CLASS A: /8 = 255.0.0.0 CLASS B: / 16 = 255.255.0.0 CLASS C: /24 = 255.255.255.0 --- Mạng ADDRESSES ![image](https://github.com/psaumur/CCNA/assets/106411237/12178b46-2604-468b-a11c-2a94087b023d) If the HOST PORTION của một Địa chỉ IP là ALL 0's, it means it là the Mạng Địa chỉ = the identifier của the Mạng itself. Example: 192.168.1.0/24 = THIS là một Mạng Địa chỉ. A Mạng Địa chỉ cannot be assigned đến một HOST. A Mạng Địa chỉ là the FIRST Địa chỉ. ![image](https://github.com/psaumur/CCNA/assets/106411237/53eafb43-2a6f-422c-af19-866946d78efa) If the HOST PORTION của một Địa chỉ IP là ALL 1's, it means it là the Broadcast Địa chỉ cho the Mạng. A Broadcast Địa chỉ cannot be assigned đến một HOST. DESTINATION IP : 192.168.1.255 (Broadcast Địa chỉ IP) DESTINATION MAC : FFFF.FFFF.FFFF (Broadcast Địa chỉ MAC) Because của the two 'reserved' addresses, the range của USABLE HOST ADDRESSES là 1 đến 254.
