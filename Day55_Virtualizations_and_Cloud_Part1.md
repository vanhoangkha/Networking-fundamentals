VIRTUALIZATION và CLOUD: PART 1 VIRTUAL SERVERS - Although Cisco là more known cho their networking DEVICES (ROUTERS, SWITCHES, FIREWALLS), they also offer HARDWARE SERVERS such as UCS (Unified Computing System) - The largest vendors của HARDWARE SERVERS include Dell, EMC, HPE, and IBM --- SERVERS BEFORE VIRTUALIZATION ![image](https://github.com/psaumur/CCNA/assets/106411237/365cde17-88c6-4149-91f2-d67c79590aec) - Before VIRTUALIZATION, there was một one-to-one relationship between một PHYSICAL SERVER và OPERATION SYSTEM - In that OPERATING SYSTEM, apps providing SERVICES (such as một WEB SERVER, EMAIL SERVER, etc) would run - One PHYSICAL SERVER sẽ be used cho the WEB SERVER, one cho the EMAIL SERVER, one cho the DATABASE SERVER, etc. - This là inefficient cho multiple reasons: - Each PHYSICAL SERVER là expensive và takes up space, power, etc. - The RESOURCES trên each PHYSICAL SERVER (CPU, RAM, STORAGE, NIC) are typically under-utilized --- VIRTUALIZATION (TYPE 1 HYPERVISOR) ![image](https://github.com/psaumur/CCNA/assets/106411237/62a40737-7451-4b38-a5bd-abd9367cbd40) - VIRTUALIZATION allows us đến break the one-to-one relationships của HARDWARE đến OS, allowing multiple OS’s đến run trên một single PHYSICAL SERVER - Each INSTANCE là called một VM (Virtual Machine) - A HYPERVISOR là used đến manage và allocate the HARDWARE RESOURCES (CPU, RAM, etc.) to each VM - Another name cho một HYPERVISOR là VMM (Virtual Machine Monitor) - The type của HYPERVISOR which runs directly trên top của hardware là called một TYPE 1 HYPERVISOR - Examples include : VMware ESXi, Microsoft Hyper-V, etc. - TYPE 1 HYPERVISORS là also called *bare-metal hypervisors* because they run directly trên the hardware (metal). - Another term is *native hypervisor* - This là the type của HYPERVISOR used trong data center environments --- VIRTUALIZATION (TYPE 2 HYPERVISOR) ![image](https://github.com/psaumur/CCNA/assets/106411237/25a29935-a56d-4ffe-b9e4-15c5c50bca46) - TYPE 2 HYPERVISORS run as một program trên một OS like một regular computer program - Examples: VMware Workstation, Oracle Virtualbox, etc - The OS running directly trên the hardware là called the HOST OS - The OS running trong một VM là called một GUEST OS - Another name cho một TYPE 2 HYPERVISOR is *hosted hypervisor* - Although TYPE 2 HYPERVISORS là rarely used trong data center environments, they là common trên personal-use devices (Ví dụ, if một MAC/Linux user needs đến run một app that là only supported trên Windows, or vice-versa) --- WHY VIRTUALIZATION? - PARTITIONING : - Run multiple OS’s trên ONE PHYSICAL MACHINE - Divide system resources between VIRTUAL MACHINES - ISOLATION : - Provide FAULT và Bảo mật ISOLATION tại the hardware level - Preserve performance với advanced resource controls - Đóng gói : - Save the entire state của một virtual machine đến files - Move và copy virtual machines as easily as moving và copying files - HARDWARE INDEPENDENCE : - Provision hoặc migrate any virtual machine đến any physical server ![image](https://github.com/psaumur/CCNA/assets/106411237/f0f5f886-7924-46fc-addf-6916f0d2b2c9) --- VIRTUAL NETWORKS ![image](https://github.com/psaumur/CCNA/assets/106411237/7bf3f22c-a7b8-41bf-bc1e-8c128a41f20f) - VMs là connected đến each other và the EXTERNAL Mạng via một VIRTUAL Switch running trên the HYPERVISOR - Just like một regular PHYSICAL Switch, the vSWITCH’s INTERFACES có thể operate as Truy cập PORTS hoặc Trunk PORTS và use VLANs đến separate the VMs tại LAYER 2 - INTERFACES trên the vSWITCH connect đến the PHYSICAL NIC (or NICs) of the SERVER đến communicate với the EXTERNAL Mạng --- INTRO đến CLOUD COMPUTING - Traditional IT Hạ tầng deployments were some combination của the following: - ON-PREMISES - All SERVERS, Mạng DEVICES, and other Hạ tầng là located trên company property - All equipment là purchased và owned bởi the company using it - The company là responsible cho the necessary space, power, and cooling - CO-LOCATION - Data centers that rent out space cho customers đến put their Hạ tầng (SERVERS, Mạng DEVICES) - The data center provides the space, electricity, and cooling - The SERVERS, Mạng DEVICES, etc là still the responsibility của the end customer, although they là not located trên the customer’s premises - CLOUD SERVICE provide một alternative that là hugely popular và là continuing đến grow - Most people associate “CLOUD” with PUBLIC CLOUD PROVIDERS such as AWS - Although this là the most common USE của CLOUD SERVICES, it’s not the only one --- CLOUD SERVICES - The American NIST (National Institute của Standards và Technology) defined cloud computing trong SP (Special Publication) 800-145 ![image](https://github.com/psaumur/CCNA/assets/106411237/746a4f38-01b9-49cf-8dd9-5522b4cabf7b) - To understand what the CLOUD is, let’s look tại the following outlined trong SP 800-145: - FIVE ESSENTIAL CHARACTERISTICS - THREE SERVICE MODELS - FOUR DEPLOYMENT MODELS --- THE FIVE ESSENTIAL CHARACTERISTICS của CLOUD - ON-DEMAND SELF-SERVICE - The CUSTOMER là able đến use the SERVICE (or stop the SERVICE) freely (via một web portal) without direct communication đến the SERVICE PROVIDER - BROAD Mạng Truy cập - The SERVICE là available through standard Mạng connections (ie: the Internet hoặc PRIVATE WAN) and có thể be Truy cập through many kinds của DEVICES - RESOURCE POOLING - A POOL của RESOURCES là provided bởi the SERVICE PROVIDER và when một CUSTOMER requests một SERVICE (Ví dụ creates một new VM), the RESOURCES đến fulfill that request là allocated từ the shared POOL - RAPID ELASTICITY - CUSTOMERS có thể quickly expand the SERVICE they use trong the CLOUD (Ví dụ: add new VMs, expand STORAGE, etc) from một POOL của RESOURCES that appear đến be infinite. Likewise, they có thể quickly reduce their SERVICES when not needed - MEASURED SERVICE - The CLOUD SERVICE PROVIDER measures the CUSTOMER’s usage của CLOUD RESOURCES và the CUSTOMER có thể measure their own use as well. CUSTOMERS là charged based trên usage (Ví dụ: X Dollars per Gigabyte của STORAGE per day) --- THE THREE SERVICE MODELS của THE CLOUD ![image](https://github.com/psaumur/CCNA/assets/106411237/a3f0e08a-3207-4a69-aa81-d4142d6735a3) - In CLOUD COMPUTING, everything là provided trên a “SERVICE” model - Ví dụ: rather than the END USER buying một PHYSICAL SERVER, mounting it trên một rack, installing the hypervisor, creating the VM, etc. the SERVICE PROVIDER offers all của this as một SERVICE - There là một variety của SERVICES referred đến as “___________ as một SERVICE” or “__aaS” - The THREE SERVICE MODELS của CLOUD COMPUTING are: SOFTWARE as một SERVICE (SaaS) - Example : MS Office 365 ![image](https://github.com/psaumur/CCNA/assets/106411237/5bcfedb7-3ab6-462a-a089-09884d220ab7) PLATFORM as một SERVICE (PaaS) - Examples : AWS Lambda và Google App Engine ![image](https://github.com/psaumur/CCNA/assets/106411237/e3886b6b-4ed8-4358-ba47-e2f50378c53d) Hạ tầng as một SERVICE (Iaas) - Examples: Amazon EC2 và Google Compute Engine ![image](https://github.com/psaumur/CCNA/assets/106411237/f8144a61-0d7f-4928-9e47-73fb969e0b4a) --- DEPLOYMENT MODELS - Most people assume that “CLOUD” means PUBLIC CLOUD PROVIDERS like AWS, AZURE, and GCP - Although “PUBLIC CLOUD” is the most common deployment model, it’s not the ONLY one - The FOUR DEPLOYMENT MODELS của CLOUD COMPUTING are: - PRIVATE CLOUD ![image](https://github.com/psaumur/CCNA/assets/106411237/b8953a31-3861-41ef-99df-62a49b97610a) - PRIVATE CLOUDS là generally only used bởi large enterprises - Although the CLOUD là PRIVATE, it có thể be owned bởi một THIRD PARTY - Ví dụ: AWS provides PRIVATE CLOUD SERVICES cho the American DoD - PRIVATE CLOUDS có thể be trên hoặc OFF PREMISES - Many people assume “CLOUD” and “ON-PREM” are two different things but that là not always the case - The same kind của SERVICES offered là the same as trong PUBLIC CLOUDS (SaaS, PaaS, IaaS) but the Hạ tầng là reserved cho một SINGLE ORGANIZATION - COMMUNITY CLOUD ![image](https://github.com/psaumur/CCNA/assets/106411237/1c9008e9-205b-4ca8-8236-fc0b02c3addc) - Least common CLOUD deployment - Similar đến PRIVATE CLOUD, but the Hạ tầng là reserved cho use bởi một SPECIFIC GROUP hoặc ORGANIZATION - PUBLIC CLOUD ![image](https://github.com/psaumur/CCNA/assets/106411237/94e9c895-9538-4664-93db-085f013ee9fb) - The most common CLOUD deployment - Popular PUBLIC CLOUD service providers include: - AWS - MS AZURE - GCP (Google Cloud Platform) - OCI (Oracle Cloud Hạ tầng) - IBM Cloud - Alibaba Cloud - HYBRID CLOUD ![image](https://github.com/psaumur/CCNA/assets/106411237/14f910cf-b6e2-4f75-8959-4589d2592e1c) - Basically ANY combination của the preview THREE DEPLOYMENT TYPES - Example: A PRIVATE CLOUD which có thể offload đến một PUBLIC CLOUD when necessary --- BENEFITS của CLOUD COMPUTING - COST - CapEx (Capital Expense) of buying HARDWARE và SOFTWARE, setting up DATA CENTERS, etc. are reduced hoặc eliminated - GLOBAL SCALE - CLOUD SERVICES có thể scale GLOBALLY tại một rapid pace. SERVICES có thể be set up và offered đến CUSTOMERS từ một geographic location close đến them - SPEED / AGILITY - SERVICES là provide trên DEMAND và vast amounts của RESOURCES có thể be provisioned within minutes - PRODUCTIVITY - CLOUD SERVICES remove the need cho many time-consuming tasks such as procuring physical servers, racking them, cabling, installing và updating equipment, etc. - RELIABILITY - Backups trong the CLOUD là very easy đến perform. Data có thể be mirrored tại multiple sites trong different geographic locations đến support disaster recovery --- CONNECTION đến PUBLIC CLOUDS ![image](https://github.com/psaumur/CCNA/assets/106411237/671747bb-6908-47bb-b9c8-47f2df82c821)
