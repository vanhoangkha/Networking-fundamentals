RIP và EIGRP (IGP : DYNAMIC VECTOR) Định tuyến INFORMATION Giao thức (RIP) - Định tuyến Information Giao thức (Industry Standard) - is một DISTANCE VECTOR IGP - uses Định tuyến-By-Rumor logic đến learn/share routes - Uses HOP COUNT as it’s Metric (One Router = One Hop) Bandwidth là irrelevant - MAX HOP COUNT là 15 (anything more là considered unreachable) - Has THREE VERSIONS: - RIPv1 và RIPv2; used cho IPv4 - RIPng (RIP Next Generation) used cho IPv6 - Uses TWO MESSAGE TYPES: - REQUEST : - To ask RIP-ENABLED neighbour ROUTERS đến send their Định tuyến TABLE - RESPONSE: - To SEND the LOCAL Router’s Định tuyến TABLE đến neighbouring ROUTERS By Default, RIP-Enabled ROUTERS sẽ share their Định tuyến TABLE every 30 seconds RIPv1 và RIPv2 RIPv1: - Only advertises *classful addresses* (Class A, Class B, Class C) - Doesn’t support VLSM, CIDR - Doesn’t include Mạng con MASK information trong ADVERTISEMENTS (RESPONSE messages) - Example: - 10.1.1.0/24 sẽ become 10.0.0.0 (Class một Địa chỉ, so assumed đến be /8) - 172.16.192.0/18 sẽ become 172.16.0.0 (Class B Địa chỉ, so assumed đến be /16) - 192.168.1.40/30 sẽ become 172.168.1.0 (Class C Địa chỉ, so assumed đến be /24) - Messages là Broadcast đến 255.255.255.255 RIPv2: - Supports VLSM, CIDR - Includes Mạng con MASK information trong ADVERTISEMENTS - Messages are **Multicast** to 224.0.0.9 - Broadcast Messages là delivered đến ALL devices trên the local Mạng - Multicast Messages là delivered only đến devices đến have joined that specific ***Multicast group*** --- CONFIGURING RIP ![image](https://github.com/psaumur/CCNA/assets/106411237/1d14ec8b-121c-4666-b608-1e5d1889424c) The **“Mạng”** Lệnh tells the Router to: - Look cho INTERFACES với một Địa chỉ IP that là trong the specific RANGE - ACTIVATES RIP trên the INTERFACES that fall trong the RANGE - Form ADJACENCIES với connected RIP neighbors - Advertise the **Mạng PREFIX của the Giao diện** (NOT the prefix trong the “Mạng” Lệnh) The OSPF và EIGRP **“Mạng”** commands operate trong the same way Because the RIP “Mạng” Lệnh là CLASSFUL. It sẽ automatically convert đến CLASSFUL networks - 10.0.0.0 là assumed đến be 10.0.0.0/8 - R1 sẽ look cho ANY INTERFACES với một Địa chỉ IP that matches 10.0.0.0/8 (because it is /8 it only needs đến match the FIRST 8 bits) - 10.0.12.1 và 10.0.13.1 both match SO RIP là ACTIVATED trên G0/0 và G0/1 - R1 then forms ADJACENCIES với its neighbors R2 và R3 - R1 ADVERTISES 10.0.12.0/30 và 10.0.13.0/30 (NOT 10.0.0.0/8) to it’s RIP neighbors ![image](https://github.com/psaumur/CCNA/assets/106411237/2a9452f0-b48f-499d-938f-0a3db5ff6587) - Because the “Mạng” Lệnh là CLASSFUL, 172.16.0.0 là assumed đến be 172.16.0.0/16 - R1 sẽ look cho ANY INTERFACES that match 172.16.0.0/16 - 172.16.1.14 matches, so R1 sẽ ACTIVATE RIP trên G2/0 - There là NO RIP neighbors connected đến G2/0 so no NEW ADJACENCIES là formed - Although there là NO RIP neighbors, R1 sẽ still send ADVERTISEMENTS out của G2/0. - This là unnecessary traffic, so G2/0 nên be configured as a **passive Giao diện** ![image](https://github.com/psaumur/CCNA/assets/106411237/634f4c6b-291c-4a21-8ae2-c8283044efce) - the “passive-Giao diện” Lệnh tells the Router đến stop sending RIP advertisements out của the specified Giao diện (G2/0) - However, the Router sẽ continue đến ADVERTISE the Mạng prefix của the Giao diện (172.16.1.0/28) to it’s RIP neighbors (R2, R3) - You nên ALWAYS use this Lệnh trên INTERFACES which don’t have any RIP neighbors - EIGRP và OSPF both have the same passive Giao diện functionality, using the same Lệnh. --- Cách ADVERTISE một Default Tuyến đường INTO RIP ![image](https://github.com/psaumur/CCNA/assets/106411237/57de003e-0e8e-48c7-bb72-fbe25208d847) ![image](https://github.com/psaumur/CCNA/assets/106411237/1c500efd-e96b-4e49-b1f4-f99c54b0e877) To SHARE this Default Tuyến đường với R1’s RIP neighbors, using this Lệnh: ![image](https://github.com/psaumur/CCNA/assets/106411237/799d818a-06cc-4f29-8c74-c67639c9d014) RIP doesn’t care about Giao diện AD cost (RIP cost là 120), only “hops”. Since both have một equal number of “hops”, both paths appear trong the Default Tuyến đường (Gateway của Last Resort) ![image](https://github.com/psaumur/CCNA/assets/106411237/1deccb54-02e0-4d3b-b203-277d656504b3) --- “show ip protocols” (for RIP) ![image](https://github.com/psaumur/CCNA/assets/106411237/b7ab4046-b6eb-4e19-b7eb-2c5d2889293a) “Maximum path: 4” is the Default but có thể be changed với this Lệnh: ![image](https://github.com/psaumur/CCNA/assets/106411237/35d524bd-055d-4c5e-a84b-f507a87738e0) “Distance” (AD) can be changed với this Lệnh (Default là 120) ![image](https://github.com/psaumur/CCNA/assets/106411237/5247942b-1d6b-419f-a4c7-75bfcca43fe6) --- ENHANCED INTERIOR Gateway Định tuyến Giao thức (EIGRP) - Enhanced Interior Gateway Định tuyến Giao thức - is một DISTANCE VECTOR IGP - Was Cisco proprietary, but Cisco has now published it openly so other vendor có thể implement it trên their equipment - Considered an “advanced” / “hybrid” DISTANCE VECTOR Định tuyến Giao thức - Much faster than RIP trong reacting đến changes trong the Mạng - Does NOT have the 15 ‘hop count’ limit của RIP - Sends messages using Multicast Địa chỉ **224.0.0.10 (Memorize this number)** - Is the ONLY IGP that có thể perform **unequal**-cost load-balancing (by Default, it performs ECMP load-balancing over 4 paths like RIP) --- Cấu hình của EIGRP ![image](https://github.com/psaumur/CCNA/assets/106411237/f2b42631-bcb9-4f62-afe9-b7bb1e7e0d7e) “Router EIGRP <Autonomous System number>” - The AS (Autonomous System) number phải MATCH between ROUTERS hoặc they sẽ NOT form một ADJACENCY và share Tuyến đường information - Auto-summary có thể be ENABLED hoặc DISABLED bởi Default; depending trên the Router/IOS version. If ENABLED, Vô hiệu hóa it. - The **“Mạng”** Lệnh sẽ assume một CLASSFUL Địa chỉ, if you don’t specify the Mạng con MASK - EIGRP uses a *wildcard mask* instead của một regular Mạng con mask A WILDCARD MASK là an “inverted” Mạng con MASK - All 1’s trong the Mạng con MASK là 0 trong the equivalent WILDCARD MASK. - All 0s trong the Mạng con MASK là 1 trong the equivalent WILDCARD MASK. ![image](https://github.com/psaumur/CCNA/assets/106411237/f64e06d3-75ad-4f4f-b7d6-26f27ffae541) “0” in the WILDCARD MASK = BITS phải MATCH ! “1” in the WILDCARD MASK = Do not have đến match ![image](https://github.com/psaumur/CCNA/assets/106411237/13130e3c-de62-4f80-9c7d-256a2ed47e74) ![image](https://github.com/psaumur/CCNA/assets/106411237/1aa2cd2c-397f-4f3b-86ed-81eddf2677a6) ![image](https://github.com/psaumur/CCNA/assets/106411237/500ac3b0-5d83-4691-ab94-06fd330a9111) --- “show ip protocols” (for EIGRP) ![image](https://github.com/psaumur/CCNA/assets/106411237/f3f169da-d733-4da9-8d8a-c90e2077d8a7) “Router ID” Router ID order của priority: - Manual Cấu hình - Highest Địa chỉ IP trên một LOOPBACK Giao diện - Highest Địa chỉ IP trên một PHYSICAL Giao diện ![image](https://github.com/psaumur/CCNA/assets/106411237/29757624-9e79-4878-8724-36d5da43f39b) “Distance” (AD) EIGRP has TWO VALUES: - Internal = 90 - External = 170 MEMORIZE THESE VALUES! “show ip Tuyến đường” (for EIGRP) ![image](https://github.com/psaumur/CCNA/assets/106411237/8216ceb6-0d3f-42e7-8e5b-46e810097fb8) NOTE the large Metric numbers. This là một DOWNSIDE đến EIGRP - even trên small networks! --- EIGRP Metric - By Default, EIGRP uses BANDWIDTH và DELAY đến calculate Metric - Default “K” values are: - K1 = 1, K2 = 0, K3 = 1, K4 = 0, K5 = 0 💡 Simplified calculation : Metric = BANDWIDTH (Slowest Link) + DELAY (of ALL LINKS) --- EIGRP TERMINOLOGY - **Feasible Distance** = This Router’s Metric value đến the Tuyến đường’s DESTINATION - **Reported Distance** (aka **Advertised Distance**) = The neighbor’s Metric value đến the Tuyến đường’s DESTINATION ![image](https://github.com/psaumur/CCNA/assets/106411237/436ba2c2-43e7-4fea-a527-f88a8e4460bc) - **Successor =** the Tuyến đường với the LOWEST Metric đến the DESTINATION (the best Tuyến đường) - **Feasible Successor** = An alternate Tuyến đường đến the DESTINATION (not the best Tuyến đường) which meets the *feasibility condition* **FEASIBILITY CONDITION** : A Tuyến đường là considered a ***Feasible Successor*** if it’s ***Reported Distance*** is LOWER than the Successor Tuyến đường’s ***Feasible distance*** ![image](https://github.com/psaumur/CCNA/assets/106411237/206db633-3a7e-4d11-bb80-029ea8107503) --- EIGRP : UNEQUAL-COST LOAD-BALANCED ![image](https://github.com/psaumur/CCNA/assets/106411237/23a2045b-a925-4f75-b0f8-78cbae2aa1e2) “maximum Metric variance 1” = the Default value Variance 1 = only ECMP (Equal-Cost Multiple Path) load-balancing sẽ be performed ![image](https://github.com/psaumur/CCNA/assets/106411237/824dac1d-38dc-4e7e-bb48-b382918230ff) Variance 2 = ***feasible successor*** routes với một FD up đến 2x the ***successor*** Tuyến đường’s FD có thể be used đến load-balance 💡 EIGRP sẽ only perform UNEQUAL-COST LOAD-BALANCING over ***feasible successor*** ROUTES. If một Tuyến đường doesn’t meet the ***feasibility condition***, it sẽ NEVER be selected cho load-balancing, regardless of **variance**
